## 问题诊断

在当前窄视口（836px CSS 宽度）下：

- **底部翻页控件**：位于 `absolute bottom-4 left-1/2 -translate-x-1/2`（屏幕底部水平居中），宽度约 200px。
- **编辑模式浮动条 / 工具面板**：位于 `fixed bottom-4 right-4`，宽度约 320–360px。

两者都贴在底边，且居中条会向右延伸到右半屏，于是和右下角的编辑工具栏发生重叠 —— 用户无法点击到翻页箭头/页码。

## 修复方案

把编辑模式下的浮动控件挪到**屏幕右上角**（不会与任何已有 UI 重叠：顶部品牌条在 slide 内部缩放层里，不在 viewport 顶部），仅修改定位，不改任何功能/交互逻辑。

### 文件改动：`src/components/editor/EditorPanel.tsx`

1. **第 153 行**（编辑模式浮动小按钮条）  
   将 `fixed bottom-4 right-4` 改为 `fixed top-4 right-4`。

2. **第 178 行**（"编辑工具" 设置面板，含导出/导入/重置/退出）  
   将 `fixed bottom-4 right-4` 改为 `fixed top-4 right-4`。

3. **选中文字/图片时弹出的属性面板**：  
   - 已有 `computeAutoPos` 智能避让逻辑（自动放在所选元素右侧/左侧/上下方），无需改动。  
   - 兜底 style 为 `{ right: 16, top: 80 }`，本来就在右上角，无需改动。

### 不改动

- 翻页按钮、页码控件、键盘快捷键、保存逻辑、上传逻辑均保持不变。
- 仅 2 处 className 调整，纯 UI 位置修复。

### 验证点

- 编辑模式下，底部翻页按钮可正常点击，不再被遮挡。
- 右上角浮动条与顶部 BRAND BOOK 标题（在 slide 缩放层内）不冲突。
- 拖动选中元素属性面板的功能（onDragStart）不受影响。
