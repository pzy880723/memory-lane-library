## Plan

将导出改成“内存下载”而不是“页面跳转下载”，避免 Lovable 预览环境在点击 PDF/PPT 时离开当前预览页，从而触发你看到的真实登录失效/重新鉴权页面。

### 要做的修改

1. 重写导出方法
- 把 `src/lib/export.tsx` 从当前的 `<a href="/exports/...">` 直接点击下载，改成：
  - `fetch('/exports/xxx')`
  - 转成 `blob`
  - `URL.createObjectURL(blob)`
  - 再用临时 `<a download>` 触发保存
- 下载完成后及时 `revokeObjectURL`，避免内存泄漏。

2. 给导出流程加稳定兜底
- 如果 `fetch` 失败，返回明确错误，不做当前页跳转。
- 这样即使预览环境异常，也不会把当前页面带离开应用。

3. 调整首页导出交互
- 在 `src/pages/Index.tsx` 里把导出按钮处理改成异步。
- 点击时显示“正在准备下载”，成功后提示“下载已开始”，失败时提示“下载失败，请重试”。
- 导出进行中临时禁用重复点击，避免用户连点触发多次请求。

4. 保持现有“预生成文件”方案不变
- 继续使用构建时生成好的 `public/exports/*.pdf` 和 `*.pptx`。
- 只改“下载方式”，不改你已经确认的“预生成、秒下、网页一致”这套方案。

### 预期结果

- 点击下载 PDF/PPT 时，当前 SPA 不会被跳走。
- Lovable 预览不会再切回登录页或重新鉴权页。
- 刷新时如果你仍停留在当前应用页面，不会因为刚才下载动作导致被带离系统。
- 你自己部署到正式服务器后也会更稳，因为下载体验会统一成“直接保存”而不是“浏览器尝试打开文件页”。

## 技术细节

当前实现本质上是：

```text
用户点击下载
-> a.href=/exports/file.pdf
-> 浏览器可能把当前预览上下文导航到文件地址
-> Lovable preview 的鉴权/iframe 状态被打断
-> 再刷新时回到登录或鉴权入口
```

计划改成：

```text
用户点击下载
-> fetch 静态文件
-> blob
-> object URL
-> 触发本地保存
-> 当前页面不导航
```

涉及文件：
- `src/lib/export.tsx`
- `src/pages/Index.tsx`

不需要改动：
- 预生成脚本
- PDF/PPT 内容本身
- 编辑器逻辑

## 验收标准

- 在 Lovable 预览里点击 PDF 下载，不跳登录页。
- 在 Lovable 预览里点击 PPT 下载，不跳登录页。
- 下载后立即刷新，仍停留在当前项目页面，而不是 Lovable 登录页。
- 已发布站点的下载仍正常可用。
